


# Code Description and Explanation of the Extension Structure


## Основные концепции

*Поставщики поиска* компоненты системы, которые интегрируют свои результаты поиска в общесистемный поиск GNOME Shell (Activities Overview).

Поставщик поиска — это компонент, который:

- Получает поисковые запросы от GNOME Shell
- Выполняет поиск в своих данных
- Возвращает результаты для отображения в UI
- Обрабатывает активацию результатов пользователем

В этои примере рассматривается реализация поставщика поиска реализованного как расширение GNOME Shell.


## Интерфейс SearchProviderInterface


### Обзор

`SearchProviderInterface` — интерфейс, реализующий поставщика поиска для расширения GNOME Shell.


### Свойства


#### `id`

~~~typescript
readonly id: string;
~~~

**Описание**: Уникальный строковой идентификатор поставщика поиска в системе.

Должен быть уникальным среди всех поставщиков, не только среди расширений.

Расширения, реализующие `SearchProviderInterface` обычно используют свой UUID.

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#L35)


#### `appInfo`

~~~typescript
readonly appInfo: Gio.AppInfo | null
~~~

**Описание**: `AppInfo` поставщика поиска.

Метаинформация из `AppInfo` используется Shell для декорации и группировки результатов поиска.

![appInfo_null.png](pics/appInfo_null.png)

Если реализация возвращает `null` - результаты поиска будут отображаться как значки вне какой либо группы.

Расширение может регистрировать и возвращать свой `AppInfo`, или возвращать фейковый.

![appInfo_fake.png](pics/appInfo_fake.png)

Если реализация возвращает действительный `AppInfo` - результаты будут сгруппированы в группу декорированную значком и заголовком полученными из метаданных предоставляемых `AppInfo`.

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#L47)


#### `canLaunchSearch`

~~~typescript
readonly canLaunchSearch: boolean
~~~

**Описание**: Управляет видимостью действия "Показать больше результатов".

![canLaunchSearch_true.png](pics/canLaunchSearch_true.png)

Если реализация возвращает `true` - Shell будет отображать действие "Показать больше результатов" если результатов больше чем возможно для одновременного отображения, или вызов `filterResults` реализации отфильтровал часть результатов.

Реализация `SearchProviderInterface` должна предоставить реализацию `launchSearch` - метод, который будет вызван при активации действие "Показать больше результатов".

![canLaunchSearch_false.png](pics/canLaunchSearch_false.png)

Если реализация возвращает `false` - Shell не будет отображать действие "Показать больше результатов".

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#L40)


### Методы


#### `getInitialResultSet()`

~~~typescript
getInitialResultSet(
    terms: string[], 
    cancellable: Gio.Cancellable
): Promise<string[]>
~~~

**Описание**: Обрабатывает запрос Shell на первоначальный поиск. Производит поиск и возвращает *строковые идентификаторы* полученных результатов.

**Параметры**:

- `terms` — массив поисковых терминов (слова из запроса пользователя)
- `cancellable` — объект для отмены операции

**Возвращает**: Promise с массивом строковых идентификаторов результатов.

Вызывается при каждом новом поисковом запросе. Реализация должна провести первоначальный поиск и вернуть *строковые идентификаторы* результатов. Примерами таких идентификаторов могут быть:
- URI на ресурсы/файлы совпадающие с поисковым запросом
- Хеши
- ID записей в БД

Порядок идентификаторов определяет порядок отображения.

Shell будет отправлять эти идентификаторы в другие методы реализации когда необходимо обратится к конкретному результату поиска.

**Важно**: Реализация **должна** прервать поиск по сигналу от объекта [`cancellable`][GCancellable].

**Замечание**: Shell не ограничивает поиск временными рамками, но ожидает что реализация остановит поиск, очистит связанные ресурсы и вернет результаты "как есть" по сигналу прерывания от объекта `cancellable`.

**Замечание**: [Документация требует](...) чтобы в ответ на прерывание поиска реализация отклоняла промис с ошибкой но, по видимому, шел не обрабатывает такую ситуацию  
https://gitlab.gnome.org/GNOME/gnome-shell/-/blob/main/js/ui/search.js?ref_type=heads#L702 (актуально для версии ...)  
Таким образом желательно всегда разрешать промис, а в случае прерывания или ошибки - с пустым массивом результатов.

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#L74)


#### `getSubsearchResultSet()`

~~~typescript
getSubsearchResultSet(
    previousIdentifiers: string[], 
    terms: string[], 
    cancellable: Gio.Cancellable
): Promise<string[]>
~~~

**Описание**: Обрабатывает запрос Shell на уточнение результатов поиска при добавлении новых поисковых терминов. Возвращает подмножество исходного набора результатов, или результат нового поиска.

**Параметры**:

- `previousIdentifiers` — идентификаторы из предыдущего поиска
- `terms` — новые поисковые термины (включая предыдущие)
- `cancellable` — объект для отмены операции

**Возвращает**: Promise с массивом идентификаторов уточненного поиска.

Этот метод вызывается для уточнения текущих результатов поиска при добавлении новых поисковых терминов и должен возвращать **подмножество исходного** набора результатов.

Реализация может использовать этот метод для оптимизации поиска, вместо запуска нового поиска, или просто передавать новые термины в `getInitialResultSet()`.

**Важно**: Реализация **должна** прервать поиск по сигналу от объекта [`cancellable`][GCancellable].

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#L202)


#### `filterResults()`

~~~typescript
filterResults(identifiers: string[], maxResults: number): string[]
~~~

**Описание**: Обрабатывает запрос Shell на ограничение количества отображаемых результатов текущего поиска.

Параметры:

- `identifiers` — полный список идентификаторов текущих результатов
- `maxResults` — желаемое максимальное количество результатов для отображения

**Возвращает**: Усеченный массив идентификаторов результатов.

Реализации могут:

- просто возвращать первые n элементов
- использовать свои собственные критерии для отбрасывания результатов
- игнорировать запрос

**Важно**: Реализация должна вернуть подмножество `identifiers`. Добавление новых или изменение существующих результатов не допустимо.

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#L225)


#### `getResultMetas()`

~~~typescript
getResultMetas(
    identifiers: string[], 
    cancellable: Gio.Cancellable
): Promise<ResultMetaInterface[]>
~~~

**Описание**: Обрабатывает запрос Shell на получение метаданных результатов для отображения в UI.

**Параметры**:

- `identifiers` — массив идентификаторов результатов
- `cancellable` — объект для отмены операции

**Возвращает**: Promise с массивом метаданных для каждого результата из `identifiers`.

Реализация должна получить и сопоставить каждому identifier как минимум

-  `id`
- `name`
- `createIcon`

Смотри [Интерфейс ResultMetaInterface](...)

**Замечание**: Shell не ограничивает временными рамками, но ожидает что реализация остановит обработку, очистит связанные ресурсы и вернет *пустой* массив по сигналу прерывания от объекта `cancellable`.

**Замечание**: [Документация требует](...) чтобы в ответ на прерывание процесса реализация отклоняла промис с ошибкой но, по видимому, шел не обрабатывает такую ситуацию  
https://gitlab.gnome.org/GNOME/gnome-shell/-/blob/main/js/ui/search.js?ref_type=heads#L230 (актуально для версии ...)  
Таким образом желательно всегда разрешать промис, а в случае прерывания или ошибки - с пустым массивом результатов.

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#L127)


#### `createResultObject()`

~~~typescript
createResultObject(meta: ResultMetaInterface): Clutter.Actor | null
~~~

**Описание**: Обрабатывает запрос Shell на получение пользовательского виджет для отображения результата.

**Параметры**:

- `meta` — метаданные результата

**Возвращает**:

- `Clutter.Actor` — пользовательский виджет (как правило `St.Icon`)
или
- `null` — использовать стандартное отображение Shell

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#L109)


#### `activateResult()`

~~~typescript
activateResult(identifier: string, terms: string[]): void
~~~

**Описание**: Обрабатывает запрос Shell на активацию результата поиска.

Параметры:

- `identifier` — идентификатор активированного результата
- `terms` — поисковые термины, которые привели к этому результату

Реализация может игнорировать этот запрос (например, если ...).

Если реализация обрабатывает этот запрос она должна выполнить соответствующее действие (открыть файл, запустить приложение и т.д.) связанное с конкретным identifier.

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#L248)


#### `launchSearch()`

~~~typescript
launchSearch(terms: string[]): void
~~~

**Описание**: Запрос на полное приложение поиска с заданными терминами. Как правило происходит в ответ на активацию действия "Показать больше результатов".

**Параметры**:

- `terms` — текущие поисковые термины

Реализация может игнорировать этот запрос или запустить соответствующее приложение/поисковый ресурс передав в него поисковые термины.

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#L281)


## Интерфейс ResultMetaInterface

**Использование в примере**: [SearchProvider.ts](src/SearchProvider.ts#L154)


### Обзор

Интерфейс реализующий метаданные результата поиска, используемые GNOME Shell для отображения.

### Свойства


#### `id`

~~~typescript
id: string
~~~

Уникальный идентификатор для результата.


#### `name`

~~~typescript
name: string
~~~

Название для результата.


#### `description`

~~~typescript
description?: string
~~~

Описание результата. Необязательно.


#### `clipboardText`

~~~typescript
clipboardText?: string
~~~

Текст для помещения в буфер обмена. Необязательно.

Если `clipboardText` предоставлен - он будет помещен в буфер обмена при активации результата параллельно с вызовом `activateResult`.

Примеры использования:

- URL веб-страницы
- Путь к файлу
- Результат вычисления (калькулятор)


#### `createIcon`

~~~typescript
createIcon: (size: number) => Clutter.Actor
~~~

Функция, возвращающая значок для результата указанного размера.

Параметры:

- `size` — размер иконки в пикселях

**Возвращает**: Clutter.Actor с иконкой.

**Замечание**: Для вычисления размера значка необходимо учитывать текущий коэффициент масштабирования в системе (`St.ThemeContext.get_for_stage(global.stage)`).

**Реализация в примере**: [SearchProvider.ts](src/SearchProvider.ts#159)



[GCancellable]: https://docs.gtk.org/gio/class.Cancellable.html
